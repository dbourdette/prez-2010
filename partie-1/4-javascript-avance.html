L'échange de données avec le serveur

Le javascript est capable de faire des requêtes vers le serveur afin de récupérer des données.

L'objet permettant cela est le XmlHttpRequest mais son implémentation diffère en fonction du navigateur.

AJAX est l'utilisation de cet objet pour charger dynamiquent du contenu.

function createXhrObject() {
  if (window.XMLHttpRequest) return new XMLHttpRequest();
 
  if (window.ActiveXObject) {
    var names = ["Msxml2.XMLHTTP.6.0", "Msxml2.XMLHTTP.3.0",
      "Msxml2.XMLHTTP", "Microsoft.XMLHTTP"];
    for(var i in names) {
      try{ return new ActiveXObject(names[i]); }
      catch(e){}
    }
  }
  return null; // non supporté
}

Une fois l'objet créé, on peut faire des requêtes vers le serveur et récupérer des données de n'importe quel format texte : plain, XML, HTML, JSON...

Les requêtes avec jquery

jquery facilite les appels au serveur en prenant en charge la compatibilité.

$.ajax({
  url: "samples/ajax-hello.json",
  success: function(msg){
    alert( "données: " + msg );
  }
});  // ne fonctionne pas sous chrome

Le JSON étant supporté nativement en javascript, il est possible d'évaluer (eval()) le résultat afin d'obtenir simplement des objets.

$.ajax({
  url: "samples/ajax-hello.json", dataType : "json",
  success: function(object){
    alert( "données: " + object.att1 );
  }
});  // ne fonctionne pas sous chrome

http://api.jquery.com/jQuery.ajax/

Reverse Ajax

Il existe 2 techniques afin d'obtenir régulièrement des données depuis le serveur :

le polling depuis le client : faire une requête vers le serveur toutes les n secondes.

le push depuis le serveur : le client reçoit régulièrement des données depuis le serveur.

Il n'existe pas de standard pour la seconde solution mais une liste de pattern dont le plus courant est le XMLHttpRequest long polling

Coté serveur, ce modèle d'échange est appelé comet

http://en.wikipedia.org/wiki/Reverse_Ajax

La sécurité

Le javascript est soumis a 2 restrictions afin de limiter les risques de piratage :

La sandbox : le javascript ne peut éxécuter qu'un nombre limité d'opérations et les interactions avec l'os sont très encadrées.

La same origin policy : le javascript d'une page web ne peut faire des appels que vers son propre domaine.

Du code malicieux venant d'un serveur tiers ne peut théoriquement pas accéder aux données personnelles (formulaires, cookies...).

Le Cross Site Scripting (XSS) est une violation de la same origin policy

http://en.wikipedia.org/wiki/Same_origin_policy

JSONP (JSON with Padding)

Il existe un pattern permettant de palier aux limitations de cross domain en javascript

Il est possible d'inclure un fichier javascript en provenance d'un autre domaine.

<script type="text/javascript" 
                     src="http://autredomain.fr/jsonp?callback=foo">
</script>

Le code présent dans le fichier est éxécuté : le problème est que le code distant n'a aucune connaissance sur la page hôte.

Le code va donc faire appel à une méthode en local afin de redonner la main à la page courante.

var serverData = { name : 'John', surname : 'Doe' }
foo(serverData);

La page hôte n'a donc plus qu'a définir la fonction foo afin d'être appelé quand le code injecté est exécuté.

Encore une fois, jquery propose un API afin de faciliter les requêtes aux serveurs en JSONP.

http://en.wikipedia.org/wiki/JSON#JSONP

JSONP Flickr

Flick est un service de partage de photos et de vidéos.

L'api de Flickr est complète et propose la possibilité de faire des requêtes via jsonp.

http://api.flickr.com/services/feeds/photos_public.gne?format=json

Avec jquery, on peut donc simplement écrire.

$.ajax({
 url: 'http://api.flickr.com/services/feeds/photos_public.gne?
  					format=json
  					&jsoncallback=displayPics',
 dataType: 'jsonp'
});

La fonction displayPics est donc appelée dès que le script de Flickr est chargé.

http://www.michaelhamrah.com/blog/2010/02/using-flickr-and-jquery-to-learn-jsonp/

JSONP Flickr

On peut définir la fonction displayPics comme suit.

function displayPics(data) {
   $("#flickr").empty();
	        	
   $("<img>")
      .attr("src", data.items[0].media.m)
      .appendTo($("#flickr"));
}
